Introduction à MS Orleans

A straightforward approach to building distributed, high-scale applications in .NET

Les origines

  • Créé par Microsoft Research en 2011
  • Open source depuis 2015 (repository Microsoft avec le plus d'activité)
  • Utilisé principalement par:
    • Produits Microsoft (Azure)
    • 343 Industries (Halo 4 et 5)
    • Multiple IoT project

Microsoft Orleans en quelques mots

  • C# distribué
  • Boite à outils pour services 'cloud'
  • Virtual Actor Model acteur virtuel distribué

C# distribué

  • Orleans runs your .NET objects on a cluster as if within a single process
  • Define .NET interfaces and classes, deploy to Azure, send requests to them

Boite à outils pour services 'cloud'

  • Encapsulates best practices for building scalable, reliable, elastic services
  • Framework for stateful near-real-time backends
  • 3-5x moins de code less et code plus simple
  • Nativement scalable

Virtual Actor Model

  • Adaptation of the Actor Model for challenges of the Cloud
  • Actors that exist eternally and never fail
evolution des architecture http://www.hpts.ws/papers/2013/Bykov.pdf https://www.microsoft.com/en-us/research/project/orleans-virtual-actors/

MS Orleans & Virtual Actor Model

Problems with Actor Model Frameworks

  • Too low level
    • App manages lifecycle of actors, exposed to distributed races
    • App has to deal with actor failures, supervision trees
    • App manages placement of actors - resource management
  • Developer has to be a distributed systems expert

Virtual Actor Model

  • L'instance d'acteur existe toujours (virtuellement)
  • Activations créée à la demande
  • Localisation transparente
  • L'instance d'acteur existe toujours (virtuellement)
  • Activations créée à la demande

Nomenclature MS Orleans

Virtual Actor => Grain

Serveur => Silo

Définition d'un grain

  • Acteur existe toujours(virtuellement ou effectivement)
  • Activation sur le cluster à la demande
  • Localisation transparente
  • Chaque grain a identifiant unique
  • Ne retourne que des objets de type Task
